# Buddhist Study Interface

A personal study interface for the Pāli Canon — semantic search, thematic study plans,
offline sutta reading, and note-taking.

> **This is an independent fork.** The content library is sourced from
> [The Open Buddhist University](https://buddhistuniversity.net) (CC BY 4.0 / open access).
> This project is not affiliated with or endorsed by OBU.

---

## What it does

| Feature | Detail |
|---|---|
| **Semantic search** | Vector search over 4,494+ curated resources (articles, books, talks, canon) |
| **4,200+ suttas offline** | Bhikkhu Sujato's English translations, available without internet |
| **166 study plans** | HDBSCAN clusters of sutta embeddings — follow a theme, track progress |
| **Note-taking** | Annotate passages, cite quotes, browse notes by nikāya / plan / sutta |
| **Sutta reader** | Prev/next within a study plan, progress bar, mark as read |

---

## Architecture

```
_content/               ← 4,494 markdown resources (Jekyll metadata)
sc-data/                ← SuttaCentral bilara translations (Sujato EN)
scripts/                ← Python utility scripts (clustering, ingestion helpers)
search/
├── ingestion/          ← Embedding pipeline → Qdrant
├── api/                ← FastAPI REST (port 8001)
├── server/             ← MCP server (for Claude Code integration)
├── data/               ← study_plans.json (generated by clustering)
├── docker-compose.yml
└── requirements.txt
frontend/               ← Next.js 14 app (port 3000)
├── src/app/            ← App Router pages
├── src/lib/            ← API client, local content, notes
├── public/content/     ← Built sutta texts + indexes
└── scripts/            ← build-local-content.mjs
```

## Setup

### 1. Clone

```bash
git clone https://github.com/Lsh0x/buddhist-uni.git
cd buddhist-uni
```

### 2. Get the sutta translations

The bilara translation data (~500 MB) is needed for the offline sutta texts.
Clone SuttaCentral's bilara data into `sc-data/`:

```bash
git clone --depth 1 https://github.com/suttacentral/sc-bilara-data \
  sc-data/sc_bilara_data
```

> Only the `translation/en/sujato/sutta/` subtree is used.
> A sparse/partial clone is fine if you want to save space.

### 3. Python environment

```bash
uv venv search/.venv --python 3.12
uv pip install --python search/.venv/bin/python -r search/requirements.txt
```

Or with plain pip:

```bash
python3 -m venv search/.venv
search/.venv/bin/pip install -r search/requirements.txt
```

### 4. Start Qdrant

```bash
cd search && docker-compose up -d
# Health check: curl localhost:6333/healthz
```

## Indexing

Run these once (or whenever content changes).

### Index the resource library (~25 seconds)

Embeds all 4,494 markdown resources from `_content/` into Qdrant:

```bash
PYTHONPATH=$(pwd) search/.venv/bin/python -m search.ingestion.ingest
# Force full re-index:  --recreate
# Test on 100 docs:     --limit 100
```

### Index suttas (~10 minutes, one-time)

Embeds 4,167 Sujato translations into a separate `buddhist_suttas` collection:

```bash
PYTHONPATH=$(pwd) search/.venv/bin/python -m search.ingestion.ingest_suttas
# One nikāya only:  --nikaya dn
# Force re-index:   --recreate
```

### Generate study plans (optional — already committed)

Runs HDBSCAN clustering on sutta embeddings to produce `search/data/study_plans.json`.
Only needed if you re-index suttas and want to regenerate clusters:

```bash
PYTHONPATH=$(pwd) search/.venv/bin/python scripts/cluster_suttas.py
PYTHONPATH=$(pwd) search/.venv/bin/python scripts/generate_study_plans.py
```

### Build local sutta texts

Converts bilara JSON segments into `frontend/public/content/texts/{id}.json`
(~4,200 files, needed for offline reading):

```bash
cd frontend
node scripts/build-local-content.mjs
```

## Running

### Search API

```bash
PYTHONPATH=$(pwd) search/.venv/bin/uvicorn search.api.main:app --port 8001 --reload
# Interactive docs: http://localhost:8001/docs
```

### Frontend

```bash
cd frontend
npm install
npm run dev
# → http://localhost:3000
```

The frontend reads `NEXT_PUBLIC_API_URL` (defaults to `http://localhost:8001`).

## Key API endpoints

| Endpoint | Description |
|---|---|
| `GET /search?q=...` | Semantic search over resources |
| `GET /suttas/search?q=...` | Semantic search over suttas |
| `GET /suttas/{id}` | Sutta metadata + cluster info |
| `GET /suttas/{id}/similar` | Vector-similar suttas |
| `GET /study-plans` | List all 166 thematic clusters |
| `GET /study-plans/{cluster_id}` | Full plan with ordered sutta list |
| `GET /courses` | 71 structured courses |
| `GET /reading-path?topic=...` | Personalised reading path |
